Why Barlow?

A slightly rounded grotesk with an industrial backbone. Its Semi Condensed width gives display headers a commanding, physical presence — the kind of font that says "we show up and handle it." Designed for California's public infrastructure — built to be trusted at a glance. The wide weight range (600–900) flexes from clean corporate headers on Staffing to bold, high-energy display on Events.

Why Source Sans 3?

Adobe's flagship open-source typeface, purpose-built for screen readability. Humanist construction adds warmth without undermining professionalism — critical for a brand that needs to feel approachable on Events and buttoned-up on Staffing. Excellent x-height, open apertures, carefully hinted for every screen from desktop to mobile.

07 — Line Measure
Paragraph Width Rules

Line length directly impacts readability.

This paragraph has no max-width constraint. On a wide monitor it stretches across the entire viewport making it very difficult to track from the end of one line back to the beginning of the next. This is a common mistake — it feels exhausting to read and people will skip it entirely.

✗ Too wide — no max-width. Hard to track lines.

This paragraph is constrained to 640px — roughly 65–75 characters per line at 17px body size. The eye can comfortably track from line to line, and the paragraph feels balanced. This is the target for all Sheepdog body text.

✓ Ideal — 640px / ~65–75 characters per line

This paragraph is at 420px. Works for captions, sidebars, and card descriptions, but fragments the reading flow for body text.

◆ Narrow — 420px. Cards and sidebars only.
i

Rule: Body text max-width = 600–680px. Headings can go wider (up to 800px for H1). On mobile, screen width is the constraint — maintain 16px+ side padding.

10 — Usage Rules
Do's and Don'ts

Hard rules. No exceptions.

Do

  • Use Barlow Semi Condensed 800 for H1 display only
  • Use standard Barlow for H2–H4, buttons, nav
  • Use Source Sans 3 for all body, captions, forms, UI
  • Maintain minimum line-heights from the scale
  • Use uppercase + letter-spacing for overlines, buttons, nav
  • Constrain body text to 600–680px max-width
  • Use mobile sizes at the 768px breakpoint
  • Limit headings to H1–H4

Don't

  • Mix Barlow weights within the same heading level
  • Use Semi Condensed for body text
  • Use Barlow below 600 or Source Sans below 300
  • Add italic Barlow to headings
  • Center-align body paragraphs
  • Substitute with Inter, Roboto, or system fonts
  • Use decorative, script, or serif fonts
  • Let body text run full-width
!

Print fallback: Barlow → Franklin Gothic for headings, Source Sans 3 → Helvetica Neue → Arial for body. Document every substitution.
